Ethernet-এর **Straight-through** এবং **Crossover** cable-এর পেছনের কারণ বুঝতে হলে আগে জানতে হবে Ethernet device-এর port দুটো আলাদা ধরনের হতে পারে:

- **MDI (Medium Dependent Interface)**
    
- **MDI-X (MDI Crossover)**
    

---

# প্রথমে TX এবং RX বুঝি

একটি Ethernet device-এর কিছু wire data **Transmit (TX)** করে, আর কিছু wire data **Receive (RX)** করে।

ধরো:

```
PC

TX ---> Data Out
RX <--- Data In
```

এখন data পাঠাতে হলে:

```
PC TX  --->  Other Device RX
PC RX  <---  Other Device TX
```

অর্থাৎ TX কে RX-এর সাথে connect হতে হবে।

---
# Use case :
## Straight-Through Cable 

Straight cable-এ pin-to-pin connection থাকে:

```
Pin 1 ---> Pin 1
Pin 2 ---> Pin 2
Pin 3 ---> Pin 3
Pin 6 ---> Pin 6
```

এটা তখন ব্যবহার করা হয় যখন এক device-এর TX pins অপর device-এর RX pins-এর সাথে already design অনুযায়ী মিলে যায়।

উদাহরণ:

```
PC (MDI) -------- Switch (MDI-X)
```

এখানে:

```
PC TX ----> Switch RX
PC RX <---- Switch TX
```

Switch-এর ভেতরেই crossover করা থাকে।

তাই cable-এ আর crossover লাগেনা।

---

## Crossover Cable

যখন দুই device একই ধরনের হয়:

```
PC -------- PC
```

অথবা

```
Switch -------- Switch
```

তখন সমস্যা হয়।

ধরো:

```
PC1 TX ----> PC2 TX
PC1 RX <---- PC2 RX
```

দুই TX একসাথে connect হয়েছে!

কেউ কারও data receive করতে পারবে না।

তাই cable-এর ভিতরে pair swap করা হয়।

```
Pin 1 ↔ Pin 3
Pin 2 ↔ Pin 6
```

ফলে:

```
PC1 TX ----> PC2 RX
PC1 RX <---- PC2 TX
```

এখন communication সম্ভব।

### Switch ↔ Switch

আমরা জানি, একটি switch-এর port-এর ভেতরে TX এবং RX আগে থেকেই cross করা থাকে (MDI-X)।

তাই যদি দুইটা switch-কে straight cable দিয়ে (পুরনো switch-এর ক্ষেত্রে) connect করি:

```text
Switch 1 (internal cross)
        |
   Straight Cable
        |
Switch 2 (internal cross)
```

তাহলে দুইবার cross হয়ে যায়।

```text
TX → RX → TX
```

ফলাফল:

```text
TX ↔ TX
RX ↔ RX
```

অর্থাৎ communication হবে না।

তাই cable-এর মধ্যে আবার একবার cross করা হয়:

```text
Switch 1 (cross)
      +
Cable (cross)
      +
Switch 2 (cross)
```

এখন মোট 3 বার cross হলো:

```text
TX → RX → TX → RX
```

অবশেষে:

```text
TX ↔ RX
```

ঠিকভাবে communication শুরু হয়।

---

### এক লাইনের intuition

**"Switch-এর ভিতরে already cross করা থাকে। তাই দুইটা switch connect করলে cable-এ আরেকবার cross দিতে হয়, যাতে শেষ পর্যন্ত TX গিয়ে RX-এ পৌঁছায়।"**

(তবে মনে রাখবে, এটা মূলত পুরনো Ethernet device-এর জন্য। আধুনিক device-এ Auto MDI-X থাকার কারণে এখন সাধারণ straight cable-ই বেশিরভাগ ক্ষেত্রে কাজ করে।)

# Visualisation

### Wrong (Straight cable between two PCs)

```text
PC1              PC2

TX ------------ TX
RX ------------ RX
```

দুইজনই কথা বলছে, কেউ শুনছে না।

---

### Correct (Crossover cable)

```text
PC1              PC2

TX ------------ RX
RX ------------ TX
```

এখন একজনের কথা অন্যজন শুনতে পারছে।

---

# কোন pins cross হয়?

10 Mbps এবং 100 Mbps Ethernet-এ:

|Function|Pins|
|---|---|
|TX+|1|
|TX−|2|
|RX+|3|
|RX−|6|

Crossover cable-এ:

```text
1 ↔ 3
2 ↔ 6
```

cross করা হয়।

---

# এখন তো crossover cable প্রায় দেখা যায় না কেন?

কারণ আধুনিক network device-এ **Auto MDI-X** থাকে।

যেমন:

- আধুনিক Switch
    
- Router
    
- NIC (Network Interface Card)
    

এগুলো নিজেই detect করে:

> "ওপাশে TX কোথায়, RX কোথায়?"

তারপর internal circuitry দিয়ে automatically crossover করে নেয়।

তাই আজকাল:

- PC ↔ PC
    
- PC ↔ Switch
    
- Switch ↔ Switch
    

প্রায় সব ক্ষেত্রেই সাধারণ straight-through cable কাজ করে।

### সহজ intuition

দুইজন মানুষকে কল্পনা করো:

- Mouth = TX
    
- Ear = RX
    

Communication-এর জন্য দরকার:

```text
Mouth → Ear
Ear   ← Mouth
```

যদি ভুল করে:

```text
Mouth → Mouth
Ear   ← Ear
```

হয়, তাহলে কেউ কারও কথা শুনতে পারবে না।

Ethernet crossover cable-এর একমাত্র উদ্দেশ্য হলো **TX pair-কে RX pair-এর সাথে মিলিয়ে দেওয়া**।
