# Step 1: Hamming Distance আসলে কী?

ধরো দুইটি bit pattern আছে:

```
A = 0000
B = 0001
```

শুধু শেষ bit আলাদা।

তাই Distance = 1

---

আরেকটি:

```
A = 0000
B = 1111
```

সব 4টা bit আলাদা।

তাই Distance = 4

---

**Definition**

দুইটি valid codeword-এর মধ্যে যতগুলো bit position আলাদা, সেটাই Hamming Distance।

---

# Step 2: কেন Distance গুরুত্বপূর্ণ?

ধরো আমরা শুধুমাত্র দুইটি valid codeword ব্যবহার করছি।

```
0000
1111
```

Distance = 4

---

Transmission এর সময় noise এসে bit change করতে পারে।

ধরো পাঠানো হলো:

```
0000
```

একটা bit নষ্ট হলো:

```
0001
```

Receiver পেল:

```
0001
```

এখন receiver দেখবে:

```
0001 থেকে 0000 = 1 bit দূরে
0001 থেকে 1111 = 3 bit দূরে
```

তাই receiver বুঝতে পারবে:

> "আসল codeword সম্ভবত 0000 ছিল"

---

এখানে Distance বেশি হওয়ায় confusion কম।

---

# Step 3: Minimum Hamming Distance কী?

বাস্তবে অনেকগুলো codeword থাকে।

ধরো

```
0000
0011
1100
1111
```

এখন সব pair এর distance বের করি।

```
0000 ↔ 0011 = 2

0000 ↔ 1100 = 2

0000 ↔ 1111 = 4
```

সবগুলোর মধ্যে সবচেয়ে ছোট distance = 2

এটাই

```
Minimum Hamming Distance (dmin)
```

---

# মূল কথা

Receiver-এর error detect/correct করার ক্ষমতা সম্পূর্ণ নির্ভর করে

```
dmin
```

এর উপর।
