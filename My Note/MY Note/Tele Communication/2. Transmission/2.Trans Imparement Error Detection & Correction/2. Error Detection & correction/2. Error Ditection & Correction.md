## 1. Error Detection
 
$$d_{min} \ge t + 1  $$
যেখানে,

- d_min = Minimum Hamming Distance
    
- (t) = যতগুলো error detect করতে চাই
    

### Backend Logic

ধরো দুইটি valid codeword-এর মধ্যে distance = 4।

```text
Codeword A --------4 bit-------- Codeword B
```

A থেকে B তে যেতে কমপক্ষে 4 bit change লাগবে।

তাহলে 1, 2, বা 3 bit error হলেও received word কোনো valid codeword হবে না।

Receiver বুঝবে:

> "কিছু একটা ভুল হয়েছে।"

তাই,

```text
Detect t error
⇒ dmin must be greater than t
⇒ dmin ≥ t + 1
```

## 2. Error Correction
$$d_{min} \ge 2t + 1  $$

যেখানে,

- (t) = যতগুলো error correct করতে চাই
    

### Backend Logic

Correction মানে শুধু error হয়েছে জানলেই হবে না,

> "কোন bit ভুল হয়েছে?" সেটাও জানতে হবে।

ধরো 1-bit correction চাই।

তাহলে প্রতিটি valid codeword-এর চারপাশে 1-bit radius-এর "safe zone" থাকবে।

```text
      1 bit zone
         |
         v

    [A]       [B]
------|---------|------
```

এই দুই safe zone কখনো overlap করা যাবে না।

Overlap হলে receiver বুঝবে না received word A থেকে এসেছে নাকি B থেকে।

তাই দুই codeword-এর distance কমপক্ষে হতে হবে:

```text
1 + 1 + 1 = 3
```

অর্থাৎ,

[  
$$d_{min} \ge 2(1)+1 = 3  $$
]

সাধারণভাবে,

[  
$$d_{min} \ge 2t+1  $$
]

---

## সবচেয়ে গুরুত্বপূর্ণ Intuition (এক লাইনে)

- **Detection:** "Error হয়েছে কি না" জানতে codeword গুলোকে অন্তত (t+1) দূরে রাখতে হবে।
    
- **Correction:** "Error কোথায় হয়েছে" জানতে codeword গুলোকে আরও বেশি দূরে, অর্থাৎ (2t+1) দূরে রাখতে হবে।

## Hamming Code

Standard Hamming Code-এর জন্য:

[  
$$d_{min}=3  $$
]

$$d_{min}=3$$

তাই,

- Detect = (3-1=2) bit error
    
- Correct = (\lfloor(3-1)/2\rfloor=1) bit error
    

এ কারণেই Hamming Code **Single Bit Error Correction (SEC)** করতে পারে এবং **Double Bit Error Detection (DED)** করতে পারে।
