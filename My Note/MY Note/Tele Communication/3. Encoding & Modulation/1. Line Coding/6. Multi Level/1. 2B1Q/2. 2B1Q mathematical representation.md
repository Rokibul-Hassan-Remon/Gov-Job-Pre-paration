আসলে **$$2^m = L^n$$** সমীকরণটা শুধু একটা matching relationship।

---

# Step 1: দুই পাশের অর্থ বুঝি

## Left Side: 2^m

Binary data থেকে আসছে।

যদি `m` bit নিই, তাহলে সম্ভাব্য combination: $$2^m$$

### উদাহরণ:

#### m = 2

```text
	00
	01
	10
	11
```

মোট combination:

```text
	2² = 4
```

combination।

---

#### m = 3

```text
000
001
010
011
100
101
110
111
```

মোট:

```text
2³ = 8
```

combination।

---

# Right Side: L^n

Signal side থেকে আসছে।

যেখানে:

- L = signal level সংখ্যা
    
- n = signal element (symbol) সংখ্যা
    

সম্ভাব্য signal pattern:
$$L^n$$

---

### উদাহরণ:
#### L = 3

মানে:

```text
-3V
-1V
+1V
+3V
```

৪টি level।

---

#### যদি n = 1 

তাহলে possible signal pattern:

```text
	-3V
	-1V
	+1V
	+3V
```

মোট:

```text
	4¹ = 4
```

pattern।

---

# Step 2: কেন 2^m = L^n ?

Communication system-এর কাজ:

```text
	Bit pattern
	      ↓
	Signal pattern
```

প্রতিটি bit pattern-এর জন্য অন্তত একটি unique signal pattern লাগবে।

তাই:

```text
	Number of bit patterns = Number of signal patterns
```

অর্থাৎ:

$$2^m = L^n$$

---

# Example: 2B1Q

`নাম`: **2B1Q** মানে:

```text
	m = 2
	n = 1
	L = 4
```

Check: $$2² = 4 = 4¹ $$
Perfect match।

---

Bit patterns:

```text
	00
	01
	10
	11
```

Signal patterns:

```text
	-3V
	-1V
	+1V
	+3V
```

এক-এক করে map করা যায়।

---

# Example: 8B6T

নাম:

```text
	8B6T
```

মানে:

```text
	m = 8 bit
	
	n = 6 ternary symbols
	
	L = 3
```

Check:

```text
	2⁸ = 256
	
	3⁶ = 729
```

এখানে:

```text
	729 > 256
```

অর্থাৎ signal pattern অনেক বেশি আছে।

তাই 729 টার মধ্যে 256 টা pattern ব্যবহার করলেই হবে।

এখানে strict equality দরকার নেই।

প্রকৃতপক্ষে শর্ত হলো:

$$L^n \ge 2^m
$$

# Intuition (খুব গুরুত্বপূর্ণ)

ধরো তুমি 8 জন ছাত্রকে seat দিতে চাও।

তাহলে:

```text
	Student = Bit Pattern
	
	Seat = Signal Pattern
```

যদি:

```text

```

Perfect।

তবুও সমস্যা নেই।

Communication-এও একই ব্যাপার।

```text
	Bit patterns ≤ Signal patterns
```

হতে হবে।

# Exam Shortcut

যদি line coding scheme-এ:

- `m` bit কে
    
- `n` signal element-এ
    
- `L` level ব্যবহার করে encode করা হয়
    

তাহলে relation:

$$L^n \ge 2^m$$

আর ideal/full utilization হলে:

$$L^n = 2^m$$

**মনে রাখার ট্রিক:**

> **2^m = "কত ধরনের data আছে"**  
> **L^n = "কত ধরনের signal বানানো যায়"**
> 
> Signal-এর সংখ্যা অন্তত Data-এর সংখ্যার সমান হতে হবে, নইলে কিছু data encode করাই সম্ভব হবে না।
