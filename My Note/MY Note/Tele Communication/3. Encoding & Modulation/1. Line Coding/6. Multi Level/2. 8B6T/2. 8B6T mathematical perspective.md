# তাহলে 256 Data Pattern কে কিভাবে Map করে?

এখানেই 8B6T-এর design আসে।

কারণ:

```text
729 > 256
```

তাই engineer-রা 729 possible ternary pattern-এর মধ্যে থেকে মাত্র 256টা pattern বেছে নেয়।

---

Conceptually:

|Data|Ternary Pattern|
|---|---|
|00000000|+0-+0-|
|00000001|++-0+-|
|00000010|0+-+0-|
|...|...|
|11111111|--+0+-|

---

এগুলো arbitrary না; এমনভাবে বেছে নেওয়া হয় যাতে:

- Synchronization ভালো থাকে

# সবচেয়ে গুরুত্বপূর্ণ Intuition

8B6T-তে:

```text
8 bit
↓
256 possibilities
```

এবং

```text
6 ternary symbols
↓
729 possibilities
```

তাই system বলে:

> "আমার কাছে 729টা signal codeword আছে, কিন্তু encode করতে মাত্র 256টা data pattern লাগবে। তাই 729টার মধ্যে 256টা ভালো codeword নির্বাচন করব।"

এটাই 8B6T-এর মূল ধারণা।

এখানে **729টা pattern manually তৈরি করা হয় না**। এগুলো আসে combinatorics থেকে:

3^6 = 729

যেমন binary-তে:

```text
8 bit
```

বললেই আমরা manually 256টা combination লিখি না, কিন্তু জানি:

2^8 = 256

ঠিক একইভাবে ternary side-এ:

```text
6 symbol
```

এবং প্রতিটি symbol-এর 3টি choice থাকায়:

```text
729
```

possible signal codeword তৈরি হয়।
