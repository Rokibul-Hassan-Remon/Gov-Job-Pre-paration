# 🔷 NRZ-I

👉 **I → Invert**

তোমার intuition একদম ঠিক:

> **"I দেখতে 1 এর মতো"**
> 
> তাই Bit = 1 দেখলেই Invert (change) করো।

---
### Memory Trick

```text
	1 → Invert
	0 → Stay
```

অথবা,

```text
	I = Invert
	I ≈ 1
	
	Therefore,
		1 → Invert
		0 → No Invert
```

# 🔷 Problems of Polar NRZ-I

| Problem                 | Why it happens?                                                                                                                     |
| ----------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| Baseline Wandering      | Long sequence of **0s** causes no transitions and keeps the signal at the same level for a long time, which may shift the baseline. |
| No Self-Synchronization | Long sequence of **0s** produces no transitions, so the receiver may lose clock synchronization.                                    |
| DC Component            | Long runs of identical levels can create a non-zero average voltage, so a DC component may still exist.                             |
| Bandwidth Efficiency    | Better than schemes with frequent transitions, but synchronization problems remain for long runs of 0s.                             |
