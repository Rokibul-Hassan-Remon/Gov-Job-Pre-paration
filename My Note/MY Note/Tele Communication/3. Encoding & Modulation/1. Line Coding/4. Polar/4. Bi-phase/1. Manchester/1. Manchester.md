# 🔷 Manchester Encoding Rule

প্রতিটি bit-এর **মাঝখানে (middle)** একটি transition অবশ্যই থাকবে।

একটি common convention:

|Bit|Transition in the middle|
|---|---|
|0|High → Low|
|1|Low → High|

(কিছু বই IEEE convention ব্যবহার করে, কিছু বই উল্টো convention ব্যবহার করে। কিন্তু মূল concept একই: **মাঝখানে transition অবশ্যই থাকবে**)

# 🔷 Problems of Manchester Line Coding

|Problem|Why it happens?|
|---|---|
|Higher Bandwidth Requirement|প্রতিটি bit-এর মাঝখানে mandatory transition থাকে, ফলে NRZ-এর তুলনায় বেশি signal changes হয় এবং বেশি bandwidth লাগে|
|Higher Signal Rate|একই data rate-এর জন্য বেশি signal transitions তৈরি হয়, তাই signal rate বেশি হয়|
|More Complexity|Encoding এবং decoding NRZ-এর তুলনায় একটু বেশি complex|

---

# 🔷 Advantages of Manchester Line Coding

|Advantage|Why it happens?|
|---|---|
|Self-Synchronization|প্রতিটি bit-এর মাঝখানে transition থাকায় receiver সহজে clock recover করতে পারে|
|No Baseline Wandering|Signal দীর্ঘ সময় একই level-এ থাকে না|
|No DC Component|Positive এবং negative levels প্রায় সমান সময় থাকে, তাই average voltage ≈ 0|

---

# 🔥 Compare with NRZ

|Problem|NRZ-L|NRZ-I|Manchester|
|---|---|---|---|
|Baseline Wandering|✔ Yes|✔ Yes (long 0s)|❌ No|
|No Self-Synchronization|✔ Yes|✔ Yes (long 0s)|❌ No|
|DC Component|✔ Yes|✔ Yes|❌ No|
|Bandwidth Requirement|Low|Low|✔ High|

---

# 🧠 Intuition

Think of Manchester like this:

```text
	NRZ-L  : Transition optional
	NRZ-I  : Transition depends on data
	Manchester : Transition mandatory
```

এটাই Manchester-এর পুরো magic।

---

# ⚡ Exam Memory Trick

|Encoding|What causes transition?|
|---|---|
|NRZ-L|Bit value determines level|
|NRZ-I|Bit 1 causes inversion|
|Manchester|Every bit forces a transition|

### One-line summary

> **Manchester solves baseline wandering, synchronization, and DC component problems by forcing a transition in the middle of every bit, but it requires more bandwidth than NRZ schemes.**
