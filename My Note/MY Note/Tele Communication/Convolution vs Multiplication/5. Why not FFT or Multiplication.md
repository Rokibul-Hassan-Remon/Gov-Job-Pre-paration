# 🔴 First Truth

👉  **Time complexity (n log n) ≠ Memory usage**

👉  Fast algorithm হলেও memory বেশি লাগতে পারে

---

# 🧠 Core Reason: FFT কীভাবে কাজ করে?

FFT করে:

```
Input → split → process → combine → Output
```

👉 এটাকে বলে:  
**Divide & Conquer**

---

# 🔥 Why memory heavy? (Main reasons)

---

# 🟢 1. Complex Numbers (Big Factor)

👉 FFT কাজ করে complex number দিয়ে:

$$a+jb$$

👉 Real signal থাকলেও:

- imaginary part store করতে হয়

---

## 🎯 Example

|Type|Memory|
|---|---|
|Real signal|n|
|FFT data|~ 2n|

👉  
**Memory প্রায় double হয়ে যায়**

---

# 🟢 2. Intermediate Stages (Butterfly operations)

FFT step-by-step process:

```
Stage 1 → Stage 2 → Stage 3 → ...
```

👉 প্রতিটা stage এ:

- temporary values store করতে হয়

---

## 🎯 Visualization

```cs
  Input 
	↓
[Temp Array 1] 
	↓
[Temp Array 2] 
	↓
  Output
```

👉  
**extra buffer লাগে**

---

# 🟢 3. In-place না হলে extra array লাগে

👉 Some FFT implementations:

- input array
- output array
- temp array

👉 Total:

```
~ 2n to 3n memory
```

---

# 🟢 4. Zero Padding (Hidden cost)

👉 Convolution via FFT করতে:

```
length = n + m - 1
```

👉 Zero padding করতে হয়

---

## 🎯 Example

- signal = 1000
- filter = 1000

👉 FFT size ≈ 2048 (next power of 2)

👉  
**Memory suddenly double**

---

# 🟢 5. Overlap Methods (Streaming case)

👉 Real system এ:

- overlap-add
- overlap-save

👉 এগুলোতে:

- buffer maintain করতে হয়

👉  
**extra memory again**

---

# 🔴 Compare with Convolution

## 🟠 Direct Convolution

👉 Memory:

```
just input + small variables
```

✔️ very low memory  
✔️ no extra buffer

---

## 🔵 FFT Method

👉 Memory:

```
input + complex + temp + padding
```

❌ high memory usage

---

# 🔥 Deep Insight (VERY IMPORTANT)

👉  
**FFT trades memory for speed**

---

# 🎯 Engineer Trade-off

|Factor|Convolution|FFT|
|---|---|---|
|Speed|slow|fast|
|Memory|low|high|
|Latency|low|high|
|Complexity|simple|complex|

---

# 🧠 Real-world Decision

## 🔴 Embedded System (low RAM)

👉 Choose:

- Convolution

---

## 🔵 Server / High-performance system

👉 Choose:

- FFT
