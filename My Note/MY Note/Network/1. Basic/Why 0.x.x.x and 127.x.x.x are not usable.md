### তাহলে 0.0.0.1, 0.0.0.2 ... 0.0.0.128 কেন ব্যবহার করা যায় না?

এর কারণ **RFC 1122** অনুযায়ী পুরো **`0.0.0.0/8`** ব্লক (অর্থাৎ `0.0.0.0` থেকে `0.255.255.255` পর্যন্ত) বিশেষ উদ্দেশ্যে সংরক্ষিত ছিল। তাই এগুলোকে সাধারণ Host IP হিসেবে assign করা হতো না।

অর্থাৎ,

```
0.0.0.00.0.0.10.0.0.2...0.255.255.255
```

সবই **special/reserved** address range-এর মধ্যে পড়ে।

এগুলো Internet-এ routable নয় এবং সাধারণ device-এর IP হিসেবে ব্যবহার করা হয় না।

---

### কিন্তু এগুলো কি Network ID?

এখানে সবচেয়ে গুরুত্বপূর্ণ বিষয় হলো—

**না।**

`0.0.0.1` **Network ID নয়**।

বরং এটি **Reserved address range**-এর অংশ।

অনেকে ভুল করে মনে করে,

> 0 দিয়ে শুরু মানেই Network ID।

এটা সঠিক নয়।

Network ID নির্ভর করে **Subnet Mask/CIDR**-এর উপর।

উদাহরণ:

```
Network: 192.168.1.0/24
```

এখানে

```
192.168.1.0
```

হচ্ছে Network ID।

আবার

```
Network: 10.0.0.0/8
```

এখানে

```
10.0.0.0
```

হচ্ছে Network ID।

এখানে 0 থাকা মানেই Network ID নয়।

---

### তাহলে 0.0.0.128?

এটাও Reserved block-এর মধ্যেই পড়ে।

এটিও সাধারণ Host IP হিসেবে assign করা যায় না।

---

# What about 127.x.x.x 
**`127.0.0.0` থেকে `127.0.0.255`-ই নয়, পুরো `127.0.0.0` থেকে `127.255.255.255` (`127.0.0.0/8`) পর্যন্ত Range সাধারণ Host IP হিসেবে ব্যবহার করা যায় না।**

---

## কেন?

পুরো `127.0.0.0/8` Range **Loopback Network** হিসেবে Reserved।

এর কাজ হলো **কম্পিউটার নিজের সাথেই যোগাযোগ করবে**।

যখন তুমি `127.0.0.1`-এ packet পাঠাও, সেটি কখনো Network Interface Card (NIC) দিয়ে বাইরে যায় না। Operating System সেটিকে নিজের কাছেই ফিরিয়ে দেয়।

```
Application      │      ▼TCP/IP Stack      │      ▼127.0.0.1      │      ▼Same Computer (Never leaves the machine)
```

---

## `127.0.0.0` কি?

`127.0.0.0` হলো Loopback Network-এর Network Address।

---

## `127.0.0.1` কি?

এটি সবচেয়ে পরিচিত **localhost** Address।

উদাহরণ:

```
ping 127.0.0.1
```

এটি পরীক্ষা করে TCP/IP Stack ঠিকমতো কাজ করছে কি না।

---

## `127.0.0.2`, `127.0.0.10`, `127.0.0.100`?

এগুলিও Loopback Address।

অর্থাৎ

- `127.0.0.2` ✅ Loopback
- `127.0.0.10` ✅ Loopback
- `127.0.0.200` ✅ Loopback
- `127.1.1.1` ✅ Loopback
- `127.255.255.254` ✅ Loopback

সবগুলোই নিজের কম্পিউটারকেই নির্দেশ করে।

---

## তাহলে কেন সবাই শুধু `127.0.0.1` ব্যবহার করে?

কারণ এটি একটি **Convention (প্রচলিত নিয়ম)**।

আসলে পুরো `127.0.0.0/8`-ই Loopback, কিন্তু প্রায় সব Operating System, Software এবং Documentation-এ `127.0.0.1`-কেই localhost হিসেবে ব্যবহার করা হয়।

---

## সংক্ষেপে

|Range|ব্যবহারযোগ্য Host IP?|ব্যবহার|
|---|---|---|
|`127.0.0.0`|❌ না|Loopback Network Address|
|`127.0.0.1`|❌ না (Network Host হিসেবে)|Localhost|
|`127.0.0.2` – `127.255.255.254`|❌ না (Network Host হিসেবে)|অন্যান্য Loopback Address|
|`127.0.0.0/8`|❌ না|সম্পূর্ণ Loopback Range|

### মনে রাখার সহজ নিয়ম

- **`0.0.0.0/8`** → Reserved (Unspecified/Special)
- **`127.0.0.0/8`** → Loopback (নিজের কম্পিউটারের জন্য)
- **`169.254.0.0/16`** → APIPA
- **`10/8`, `172.16/12`, `192.168/16`** → Private Network

এই চারটি Range-এর উদ্দেশ্য আলাদা, তাই এগুলোকে একসাথে মনে রাখলে IPv4 Addressing অনেক সহজ হয়ে যায়।
