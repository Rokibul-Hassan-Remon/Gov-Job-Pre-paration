অসাধারণ প্রশ্ন। **হ্যাঁ, তুমি 2⁵ = 32 ব্যবহার করতেই পারো।** আসলে এটি আরও fundamental পদ্ধতি। `256 - 224 = 32` এবং `2^(32-27) = 32`—দুটোই একই Block Size বের করে, শুধু শুরু করার দৃষ্টিভঙ্গি আলাদা।

চল দেখি কেন।

# তাহলে Networking বইগুলো `256 - Mask` কেন শেখায়?

কারণ এটি **শুধু শেষ Octet-এর জন্য নয়**, যেই Octet-এ subnetting হচ্ছে সেখানে দ্রুত Block Size বের করার একটি shortcut।

যেমন

|CIDR|Mask|256 - Mask|2^(Host Bits)|
|---|---|---|---|
|/24|255|1|2⁰ = 1|
|/25|128|128|2⁷ = 128|
|/26|192|64|2⁶ = 64|
|/27|224|32|2⁵ = 32|
|/28|240|16|2⁴ = 16|
|/29|248|8|2³ = 8|
|/30|252|4|2² = 4|

দেখো, প্রতিটি ক্ষেত্রেই **`256 - Mask = 2^(Host Bits)`**।

---

## কোন পদ্ধতি ব্যবহার করবে?

যেহেতু তুমি subnetting-এর ভিত্তি শিখছ, আমি **2^(Host Bits)** পদ্ধতিটাই আগে শেখার পরামর্শ দেব। কারণ এতে বোঝা যায় **কেন** Block Size 32 হলো।

`256 - Mask` হলো একই ধারণার একটি দ্রুত shortcut। যখন অনেক subnetting সমস্যা দ্রুত সমাধান করতে হবে (যেমন interview, CCNA, networking exam), তখন এটি খুব কাজে লাগে।

**অর্থাৎ:**

- **Concept বোঝার জন্য:** `2^(Host Bits)`
    
- **দ্রুত হিসাব করার জন্য:** `256 - Last Octet of Mask`
    

দুটোই সঠিক, কারণ তারা একই গণিতের দুইটি ভিন্ন রূপ।

# Case - 1:  IP = 192.168.10.0 / 20
এবার একটু সাবধানে চিন্তা করতে হবে। `/20` আর `/27`-এর মধ্যে একটি গুরুত্বপূর্ণ পার্থক্য আছে।

দেওয়া আছে:

```text
192.168.10.0/20
```

---

# Step 1: Host Bits বের করি

```text
32 - 20 = 12 Host Bits
```

এখন,

```text
2^12 = 4096
```

এর অর্থ কী?

এর মানে **একটি /20 subnet-এ মোট 4096টি IP Address থাকবে।**

# এটি **Block Size নয়**
(শেষ Octet-এর Block Size অর্থে), বরং **Subnet Size (Total Addresses)**।

---

# Step 2: Subnet Mask বের করি

`/20` এর Subnet Mask হলো : 

```text
255.255.240.0
```

Binary:

```text
11111111.11111111.11110000.00000000
```

লক্ষ্য করো,

Subnetting হচ্ছে **তৃতীয় Octet-এ** (240 আছে এখানে)।

---

# Step 3: Block Size বের করি

যে Octet-এ subnetting হয়েছে, সেই Octet-এ

```text
256 - 240 = 16
```

অর্থাৎ

**Block Size = 16 (Third Octet-এ)**

# সবচেয়ে গুরুত্বপূর্ণ বিষয়

অনেকে এখানে ভুল করে বলে:

> `2^12 = 4096`, তাই Block Size = 4096

এটি **ঠিক নয়** যদি "Block Size" বলতে তুমি Octet-এর increment বোঝাও।

এখানে দুইটি আলাদা ধারণা আছে:

|Concept|/20 এর মান|
|---|---|
|Host Bits|12|
|Total IP Addresses|2¹² = **4096**|
|Usable Hosts|4094|
|Block Size (Third Octet increment)|**16**|

## একটি সহজ নিয়ম

- যদি Subnet Mask **একই Octet-এর মধ্যে শেষ হয়** (যেমন `/25`, `/26`, `/27`), তখন `2^(Host Bits)` এবং `256 - Mask` থেকে পাওয়া মান একই সংখ্যার মতো মনে হয়।
    
- কিন্তু `/20`, `/18`, `/13`-এর মতো Prefix-এ Host Bits একাধিক Octet জুড়ে থাকে। তখন:
    
    - `2^(Host Bits)` = **মোট Address সংখ্যা**
        
    - `256 - Mask` = **যে Octet-এ subnetting হয়েছে, সেই Octet-এর Block Size (increment)**
        

এটাই `/27` এবং `/20`-এর মধ্যে মূল পার্থক্য।

এখন যদি চাও, আমি দেখাতে পারি **কেন `/24`-এর নিচে গেলে `2^HostBits` আর `256 - Mask` এক হয়, কিন্তু `/20`-এর ক্ষেত্রে আর এক হয় না**—একটি Binary চিত্র দিয়ে। এটি একবার বুঝে গেলে subnetting অনেক সহজ হয়ে যাবে।
