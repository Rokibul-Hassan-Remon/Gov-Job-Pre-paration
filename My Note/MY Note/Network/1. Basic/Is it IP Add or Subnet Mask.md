> **`255.255.255.192` কোনো IP Address নয়। এটি একটি Subnet Mask।**

একই ধরনের ৪টি সংখ্যা (dot-decimal notation) ব্যবহার হওয়ায় অনেকেই IP Address এবং Subnet Mask-কে গুলিয়ে ফেলেন।

নিচের টেবিলটি দেখো:

| Example           | এটি কী?                           | ব্যাখ্যা                           |
| ----------------- | --------------------------------- | ---------------------------------- |
| `192.168.1.100`   | IPv4 Address                      | একটি device-এর address             |
| `255.255.255.192` | Subnet Mask                       | Network ও Host অংশ আলাদা করার mask |
| `192.168.1.64`    | Network Address (যদি `/26` হয়)   | একটি network-এর শুরু               |
| `192.168.1.127`   | Broadcast Address (যদি `/26` হয়) | সেই network-এর শেষ address         |

---

## কেন `255.255.255.192` IP Address নয়?

দেখো:

```
255.255.255.192
```

Binary-তে:

```
11111111.11111111.11111111.11000000
```

এখানে একটি Pattern আছে:

```
11111111.11111111.11111111.11000000
^^^^^^^^ ^^^^^^^^ ^^^^^^^^ ^^^^^^
	Network Bits (1)       Host Bits (0)
```

Subnet Mask-এর একটি নিয়ম আছে:

- প্রথমে **সব 1**
- তারপর **সব 0**

মাঝখানে 1 এবং 0 এলোমেলোভাবে থাকতে পারবে না।

যেমন এটি একটি **valid Subnet Mask**:

```
11111111.11111111.11111111.11000000
```

কিন্তু এটি **valid Subnet Mask নয়**:

```
11111111.11111111.10111111.00000000
```

কারণ এখানে 0 আসার পরে আবার 1 এসেছে।

---

## IP Address-এর Binary কেমন হয়?

একটি IP Address-এ এমন কোনো নিয়ম নেই।

যেমন:

```
192.168.1.77
```

Binary:

```
11000000.10101000.00000001.01001101
```

এখানে 1 এবং 0 যেকোনোভাবে থাকতে পারে।

---

## যখন আমরা লিখি

```
192.168.1.77/26
```

এর মানে আসলে দুটি তথ্য একসাথে দেওয়া হচ্ছে:

```
IP Address     : 192.168.1.77
Subnet Mask    : 255.255.255.192
```

অর্থাৎ,

- **`192.168.1.77`** → এটি Host-এর IP Address।
- **`/26`** → এটি Subnet Mask-এর সংক্ষিপ্ত রূপ।
- **`255.255.255.192`** → `/26`-এর পূর্ণ Subnet Mask।

Computer এই দুইটি নিয়ে Bitwise AND করে Network Address বের করে।
