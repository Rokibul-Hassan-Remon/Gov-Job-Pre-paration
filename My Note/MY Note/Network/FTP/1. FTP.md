# FTP কী?

FTP (File Transfer Protocol) হলো একটি **Application Layer Protocol** যা TCP-এর উপর চলে।

এটি দুইটি TCP connection ব্যবহার করে:

1. **Control Connection** → Command পাঠানোর জন্য
    
2. **Data Connection** → File পাঠানোর জন্য
    

এটাই FTP-এর সবচেয়ে বড় বৈশিষ্ট্য।

---

## Step 1: Client সার্ভারের সাথে Control Connection তৈরি করে

ধরো তুমি FTP Client (যেমন FileZilla) দিয়ে সার্ভারে connect করলে।

```
Client                         Server
   |                               |
   |------ TCP Port 21 ----------->|
   |                               |
```

**Port 21** হলো Control Connection-এর জন্য।

---

## Step 2: TCP Three-way Handshake

```
Client                      Server

SYN ------------------------>

             <-------------- SYN + ACK

ACK ------------------------>
```

এখন TCP connection established।

---

## Step 3: Server Greeting

```
Server:
220 Welcome to FTP Server
```

মানে

> "স্বাগতম, আমি FTP Server."

---

## Step 4: Client Username পাঠায়

```
USER remon
```

Server

```
331 Username OK.
Need password.
```

---

## Step 5: Client Password পাঠায়

```
PASS ********
```

Server

```
230 Login Successful.
```

এখন authentication শেষ।

# কেন দ্বিতীয় Connection?

Control Connection-এ command চলতে থাকে।

```

PASS
LIST
PUT
GET
DELETE

```

কিন্তু File অনেক বড় হতে পারে।

যদি একই connection-এ সব করা হয় তাহলে command block হয়ে যাবে।

তাই FTP দুটি connection ব্যবহার করে।

# FTP-তে দুটি Mode আছে

## 1. Active Mode

```
Server creates the Data Connection
```

```
Server -------------> Client
```

এতে Client-এর firewall বা NAT সমস্যা করতে পারে।

---

## 2. Passive Mode (বর্তমানে সবচেয়ে বেশি ব্যবহৃত)

```
Client creates the Data Connection
```

```
Client -------------> Server
```

এটি NAT ও Firewall-এর সঙ্গে বেশি compatible, তাই আজকাল প্রায় সব FTP client **Passive Mode** ব্যবহার করে।

# TCP-এর উপর FTP কীভাবে Data পাঠায়?

ধরো `movie.mp4` ১ GB।

FTP সরাসরি ১ GB একবারে পাঠায় না।

```
FTP
 ↓
TCP
 ↓
IP
```

TCP এটিকে ছোট ছোট segment-এ ভাগ করে।

```
Movie

↓

Chunk 1

↓

Chunk 2

↓

Chunk 3

↓

Chunk 4

...
```

প্রতিটি segment-এর sequence number থাকে।

যদি একটি segment হারিয়ে যায়:

```
Chunk 15   ❌ Lost
```

TCP সেটি detect করে এবং **শুধু সেই segment-টিই retransmit করে**। FTP-কে এ বিষয়ে কিছু করতে হয় না; FTP শুধু একটি continuous byte stream দেখে।

---

## Layer অনুযায়ী FTP

```
+----------------------------+
| FTP (Application Layer)    |
+----------------------------+
| TCP (Transport Layer)      |
+----------------------------+
| IP (Network Layer)         |
+----------------------------+
| Ethernet / Wi-Fi           |
+----------------------------+
```

- **FTP** জানে: "ফাইল আপলোড/ডাউনলোড করতে হবে।"
    
- **TCP** জানে: "সব byte ঠিকমতো এবং সঠিক ক্রমে পৌঁছাতে হবে।"
    
- **IP** জানে: "Packet কোন destination IP-তে যাবে।"
    
- **Ethernet/Wi-Fi** জানে: "লোকাল নেটওয়ার্কে frame কীভাবে পাঠাতে হবে।"
    

এই layer-ভিত্তিক দায়িত্ব ভাগ করে নেওয়ার কারণেই FTP নিজে reliability implement করে না; সেটি পুরোপুরি TCP-এর উপর নির্ভর করে।
