যখন আপনি ব্রাউজারে **google.com** বা **example.com** লিখে Enter চাপেন, তখন আপনার কম্পিউটার আসলে জানে না সার্ভারটি কোথায় আছে। সে শুধু **domain name** জানে। সার্ভারের সাথে যোগাযোগ করতে হলে তাকে **IP address** জানতে হবে। এই IP খুঁজে বের করার পুরো প্রক্রিয়াটাই **DNS (Domain Name System)** করে।

এটি সংলাপ (dialogue) আকারে দেখাই।

---

## Scenario

আপনি ব্রাউজারে লিখলেন:

```
https://example.com
```

---

## Step 1: Browser → Operating System

**Browser:**

> আমার কাছে "example.com" আছে।  
> এর IP Address জানো?

**Operating System:**

> দেখি আগে আমার DNS Cache-এ আছে কিনা।

---

## Step 2: OS DNS Cache Check

**OS:**

> Cache-এ পেলাম না।  
> তাহলে configured DNS Resolver-এর কাছে জিজ্ঞেস করি।

(এটা সাধারণত ISP-এর DNS, Google DNS (8.8.8.8), Cloudflare DNS (1.1.1.1), ইত্যাদি।)

---

## Step 3: Client → Recursive DNS Resolver

**Client → DNS Resolver**

> ভাই, "example.com" এর IP Address কত?

**Resolver**

> এক মিনিট।  
> যদি আমার Cache-এ থাকে তাহলে এখনই বলে দিচ্ছি।

Resolver নিজের cache চেক করল।

### যদি Cache-এ থাকে

**Resolver**

> IP হচ্ছে:

```
93.184.216.34
```

**Client**

> ধন্যবাদ।

এখানেই শেষ।

---

### যদি Cache-এ না থাকে

Resolver-কে পুরো Internet ঘুরে বের করতে হবে।

---

# Step 4: Resolver → Root DNS Server

**Resolver**

> Root Server,  
> example.com কোথায় পাব?

**Root Server**

> আমি exact IP জানি না।  
> কিন্তু `.com` কে কে manage করে সেটা জানি।

Root Server উত্তর দিল:

```
Go ask .com TLD Server.
```

---

# Step 5: Resolver → .com TLD Server

**Resolver**

> .com Server,  
> example.com কোথায়?

**TLD Server**

> আমি exact IP জানি না।  
> তবে example.com-এর Authoritative DNS Server হলো:

```
ns1.example.com
```

---

# Step 6: Resolver → Authoritative DNS Server

**Resolver**

> ns1.example.com,  
> example.com এর IP কত?

**Authoritative Server**

> IP হচ্ছে

```
93.184.216.34
```

---

# Step 7: Resolver → Client

Resolver এই উত্তর Cache-এ রেখে দিল।

**Resolver**

> এই নাও

```
example.com
=
93.184.216.34
```

---

# Step 8: Client → Browser

**OS**

> Browser,  
> IP পেয়ে গেছি।

```
93.184.216.34
```

---

# Step 9: Browser → Server

এখন Browser DNS-এর কাজ শেষ।

Browser TCP Connection শুরু করবে।

```
Browser
      │
      │ TCP SYN
      ▼
93.184.216.34
```

তারপর TLS Handshake (HTTPS হলে)

তারপর

```
GET /
```

পাঠাবে।

# পুরো Conversation একসাথে

```
Browser:
    example.com এর IP চাই।

OS:
    Cache দেখি...

OS:
    নাই।

OS:
    DNS Resolver, IP জানো?

Resolver:
    Cache দেখি...

Resolver:
    নাই।

Resolver:
    Root Server, example.com কোথায়?

Root:
    .com Server-এর কাছে যাও।

Resolver:
    .com Server, example.com কোথায়?

TLD (Top Level Domain) Server:
    ns1.example.com এর কাছে যাও।

Resolver:
    ns1.example.com,
    example.com এর IP কত?

Authoritative:
    93.184.216.34

Resolver:
    ধন্যবাদ।
    Cache-এ রাখলাম।

Resolver:
    Client, IP হলো
    93.184.216.34

Browser:
    ঠিক আছে।
    এখন TCP Connection শুরু করছি।
```

```text
User
 │
 ▼

 ▼

 │
 ▼

 │
 ▼

 │

 │
```

## মনে রাখার মতো বিষয়

- **Browser** ডোমেইন নাম জানে, IP জানে না।
    
- **DNS Resolver** হলো "মধ্যস্থ" যে আপনার হয়ে IP খুঁজে আনে।
    
- **Root DNS** বলে কোন **TLD**-এর কাছে যেতে হবে।
    
- **TLD DNS** বলে কোন **Authoritative DNS Server** সেই ডোমেইনের তথ্য রাখে।
    
- **Authoritative DNS Server**-ই ডোমেইনের আসল DNS record (যেমন **A**, **AAAA**, **CNAME**) থেকে IP address দেয়।
    
- IP পাওয়ার পর DNS-এর কাজ শেষ; এরপর ব্রাউজার সেই IP ব্যবহার করে TCP/TLS সংযোগ স্থাপন করে ও HTTP/HTTPS অনুরোধ পাঠায়।
