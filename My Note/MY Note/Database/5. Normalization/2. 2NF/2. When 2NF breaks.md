# ⚠️ এখন 2NF কেন break হয়?

## Rule না—real reason:

> একই information বারবার store করতে হচ্ছে, কারণ তুমি দুইটা unrelated dependency এক table-এ ঢুকিয়ে ফেলেছো।

# 🧠 Simple mental model (VERY IMPORTANT)

## 2NF violation = "wrong mixing"

```text
Student Info + Course Info + Enrollment Info
```

এক table-এ ঢুকে গেছে

---

# 🎯 Perfect intuition sentence

> 2NF breaks when we store facts that depend only on part of a composite key, because it causes duplication of independent information across multiple rows, leading to redundancy and anomalies.

---

# 🧩 Easy mental map

```text
StudentId → StudentName   (Student entity fact)
CourseId → CourseName     (Course entity fact)

Enrollment = relationship only

But table contains all 3 mixed → 2NF broken
```

---

# 🧠 One-line feeling (remember this)

> 2NF breaks when "one fact is repeated because another unrelated part of the key is changing"

---

# 🔥 Final intuition (super important)

👉 2NF is NOT about rule  
👉 It is about **avoiding “unnecessary repetition caused by mixing entities”**
