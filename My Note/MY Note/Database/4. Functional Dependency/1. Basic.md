**Functional Dependency (FD)** হলো Relational Database Theory-এর একটি খুব গুরুত্বপূর্ণ ধারণা।

সহজ ভাষায় :

> যদি একটি attribute-এর value জানা থাকলে অন্য একটি attribute-এর value uniquely নির্ধারণ করা যায়, তাহলে বলা হয় প্রথম attribute দ্বিতীয় attribute-এর উপর functionally determines করে।

Notation:

```text
A → B
```

পড়া হয় :

> A functionally determines B

অথবা ,

> B is functionally dependent on A

---

## Example 1

|StudentId|StudentName|
|---|---|
|101|Remon|
|102|Rahim|
|103|Karim|

এখানে:

```text
	StudentId → StudentName
```

কারণ:

- `StudentId = 101` হলে Name অবশ্যই **Remon**
    
- `StudentId = 102` হলে Name অবশ্যই **Rahim**
    

অর্থাৎ **StudentId** জানলেই **StudentName** জানা যায়।

## উল্টোটা কি সত্য?

ধরুন:

|StudentId|StudentName|
|---|---|
|101|Rahim|
|102|Rahim|

এখন:

```text
	StudentName → StudentId
```

❌ সত্য নয়।

কারণ "Rahim" থেকে কোন StudentId বোঝাচ্ছে তা নিশ্চিতভাবে জানা যাচ্ছে না।

---

## Candidate Key-এর সাথে সম্পর্ক

ধরুন:

| StudentId | Name  | Email             |
| --------- | ----- | ----------------- |
| 101       | Remon | remon@gmail . com |

এখানে:

```text
	StudentId → Name, Email
```

তাহলে StudentId অন্য সব attribute নির্ধারণ করতে পারছে।

এজন্য StudentId Candidate Key হতে পারে।

## কেন Functional Dependency গুরুত্বপূর্ণ?

Normalization (**1NF, 2NF, 3NF, BCNF**) পুরোপুরি Functional Dependency-এর উপর ভিত্তি করে।

যেমন:

```text
	StudentId → DepartmentId
	DepartmentId → DepartmentName
```

এখানে:

```text
	StudentId → DepartmentName
```

একটি **Transitive Dependency** তৈরি হচ্ছে।

এই ধরনের dependency খুঁজে বের করেই Normalization করা হয়।

---

## Viva Definition

> A Functional Dependency is a relationship between attributes in a relation where the value of one attribute (or set of attributes) uniquely determines the value of another attribute.

Notation:

```text
	X → Y
```

মানে:

> If two rows have the same value of X, they must also have the same value of Y.

এটাই Functional Dependency-এর সবচেয়ে গুরুত্বপূর্ণ এবং পরীক্ষামূলক সংজ্ঞা।
