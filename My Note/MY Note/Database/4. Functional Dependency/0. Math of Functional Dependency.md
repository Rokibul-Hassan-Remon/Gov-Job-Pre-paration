DBMS-এর Relational Model মূলত দুইটি গণিতের শাখার উপর দাঁড়িয়ে আছে:

1. **Set Theory** (Relation, Tuple, Attribute, Union, Intersection, Join ইত্যাদি)
    
2. **Mathematical Functions & Predicate Logic**
    

আর **Functional Dependency** এসেছে মূলত **Mathematical Function** ধারণা থেকে।

---

## Mathematics-এ Function

আমরা জানি:

```text
f(x) = y
```

একটি Function-এর Rule হলো:

> একটি input-এর জন্য মাত্র একটি output থাকবে।

উদাহরণ:

```text
StudentId → StudentName
```

|StudentId|StudentName|
|---|---|
|101|Remon|
|102|Rahim|

এখানে StudentId-কে input ধরলে:

```text
101 → Remon
102 → Rahim
```

একটি StudentId থেকে একটিই Name পাওয়া যাচ্ছে।

এটা অনেকটা Mathematical Function-এর মতো।

---

## Functional Dependency-এর Mathematical Meaning

যখন আমরা লিখি:

```text
StudentId → StudentName
```

এর মানে:

> StudentId-এর value StudentName-এর value uniquely determine করে।

এটা Function-এর সেই property-এরই database version।

---

## Set Theory-এর সাথে সম্পর্ক

Relation (Table) কে Relational Theory-তে একটি Set হিসেবে ধরা হয়।

ধরুন:

```text
	Student
	(
	  StudentId,
	  StudentName
	)
```

এটি আসলে tuples-এর একটি set:

```text
	{
	 (101, Remon),
	 (102, Rahim),
	 (103, Karim)
	}
```

Functional Dependency এই set-এর উপর একটি constraint আরোপ করে:

> একই StudentId থাকলে Name একই হতে হবে।

---

## কেন Functional Dependency এত গুরুত্বপূর্ণ?

কারণ **Candidate Key** - এর সংজ্ঞাই Functional Dependency-এর উপর ভিত্তি করে।

ধরুন:

| StudentId | Name  | Email             |
| --------- | ----- | ----------------- |
| 101       | Remon | remon@gmail . com |
|           |       |                   |

যদি:

```text
StudentId → Name, Email
```

হয়,

তাহলে StudentId relation-এর সব attribute determine করতে পারে।

এজন্য StudentId Candidate Key হতে পারে।

---

## Normalization-এ ব্যবহার

2NF, 3NF, BCNF—সবই Functional Dependency বিশ্লেষণ করে।

যেমন:

```text
StudentId → DeptId
DeptId → DeptName
```

তাহলে:

```text
StudentId → DeptName
```

এটি একটি Transitive Dependency।

3NF এই ধরনের dependency দূর করার চেষ্টা করে।

---

## সবচেয়ে গভীরভাবে বললে

Relational Theory-তে:

- **Relation** এসেছে Set Theory থেকে।
    
- **Functional Dependency** এসেছে Mathematical Function-এর "unique determination" ধারণা থেকে।
    
- **Normalization** এসেছে Functional Dependency বিশ্লেষণ থেকে।
    

অর্থাৎ:

```text
Set Theory
    ↓
Relation (Table)

Mathematical Function
    ↓
Functional Dependency

Functional Dependency
    ↓
Normalization
    ↓
Good Database Design
```

এ কারণেই DBMS-এর Normalization chapter বুঝতে হলে Function-এর ধারণা (একটি input → একটি নির্দিষ্ট output) ভালোভাবে বুঝতে হয়। Functional Dependency আসলে Mathematical Function-এর database-oriented রূপ।
