# ১. Functional Dependency (FD)

Functional Dependency (FD) হলো টেবিলের দুটি attribute-এর মধ্যে সম্পর্ক।

**X → Y** এর অর্থ:

> যদি দুইটি row-এর X এর মান একই হয়, তাহলে Y এর মানও একই হতে হবে।

### Explanation

যদি দুইটি tuple (row) `t1` এবং `t2` এর জন্য:

```
t1[X] = t2[X]
```

হয়,

তাহলে অবশ্যই:

```
t1[Y] = t2[Y]
```

হতে হবে।

তখন বলা যাবে:

```
X → Y
```
### উদাহরণ

**RollNo → Name**

|RollNo|Name|
|---|---|
|101|Rafi|
|102|Kabir|

একটি RollNo শুধুমাত্র একটি Name নির্ধারণ করে।

---

# ২. Functional Dependency-এর ধরন

## (ক) Trivial FD

যদি ডানপাশের attribute বামপাশের অংশ হয়।

**নিয়ম:**

```
	Y ⊆ X
```

উদাহরণ:

```
	{RollNo, Name} → RollNo
```

এটি Trivial FD।

### 📌 কেন এটাকে "Trivial" বলা হয়?

কারণ এটি থেকে **নতুন কোনো information বের হয় না**।

👉 আগে থেকেই জানা থাকে LHS-এর ভিতরে RHS আছে  
👉 তাই এটা খুব “obvious / obvious truth” → অর্থাৎ trivial

---

## (খ) Non-Trivial FD

যদি ডানপাশের attribute বামপাশে না থাকে।

উদাহরণ:

```
	RollNo → Name
	
	Dept → DeptHead
```

এগুলো Non-Trivial FD।

---

### 📌 Opposite (Important)

- **Trivial Dependency** → তথ্য নতুন কিছু দেয় না
	
- **Non-trivial Dependency** → নতুন তথ্য দেয়, গুরুত্বপূর্ণ
	
---

## (গ) Partial Dependency

যখন Composite Key-এর শুধুমাত্র একটি অংশ কোনো Non-prime attribute নির্ধারণ করে।

### উদাহরণ

Relation:

|RollNo|CourseID|CourseName|
|---|---|---|

Candidate Key:

```
	(RollNo, CourseID)
```

FD:

```
	CourseID → CourseName
```

এখানে পুরো key নয়, key-এর একটি অংশ `CourseID` দ্বারা `CourseName` নির্ধারিত হচ্ছে।

এটি **Partial Dependency**।

### গুরুত্বপূর্ণ

**Partial Dependency → 2NF ভঙ্গ করে।**

---

## (ঘ) Transitive Dependency

যদি:

```
X → Y
Y → Z
```

তাহলে:

```
X → Z
```

এটিই Transitive Dependency।

### উদাহরণ

```
RollNo → Dept
Dept → DeptHead
```

তাহলে,

```
RollNo → DeptHead
```

এটি Transitive Dependency।

### গুরুত্বপূর্ণ

**Transitive Dependency → 3NF ভঙ্গ করে।**

---

# ৩. Armstrong's Axioms (অবশ্যই মুখস্থ)

## ১. Reflexivity

```
Y ⊆ X
⇒ X → Y
```

উদাহরণ:

```
(RollNo, Name) → RollNo
```

---

## ২. Augmentation

```
X → Y
⇒ XZ → YZ
```

উদাহরণ:

```
RollNo → Name

⇒

(RollNo, Dept) → (Name, Dept)
```

---

## ৩. Transitivity

```
X → Y
Y → Z

⇒ X → Z
```

উদাহরণ:

```
RollNo → Dept
Dept → DeptHead

⇒ RollNo → DeptHead
```

---

## ৪. Union Rule

```
X → Y
X → Z

⇒ X → YZ
```

উদাহরণ:

```
RollNo → Name
RollNo → Dept

⇒ RollNo → Name, Dept
```

---

## ৫. Decomposition Rule

```
X → YZ

⇒ X → Y
⇒ X → Z
```

উদাহরণ:

```
RollNo → Name, Dept

⇒ RollNo → Name
⇒ RollNo → Dept
```

---

## ৬. Pseudotransitivity

```
X → Y
YZ → W

⇒ XZ → W
```

উদাহরণ:

```
RollNo → Dept
Dept, CourseID → DeptHead

⇒ RollNo, CourseID → DeptHead
```

---

# ৪. Attribute Closure (খুব গুরুত্বপূর্ণ)

Attribute Closure ব্যবহার করা হয়:

- Candidate Key বের করতে
    
- Super Key বের করতে
    
- FD যাচাই করতে
    

Attribute Closure লিখি:

```
X+
```

অর্থ:

> X থেকে Functional Dependency ব্যবহার করে যত attribute পাওয়া যায় তাদের সেট।

---

# Closure বের করার ধাপ

### Step 1

```
X+ = X
```

### Step 2

FD গুলো প্রয়োগ করো।

যদি FD-এর Left Side, X+ এর মধ্যে থাকে, তাহলে Right Side যোগ করো।

### Step 3

নতুন কিছু যোগ না হওয়া পর্যন্ত চালিয়ে যাও।

---

# গুরুত্বপূর্ণ উদাহরণ

Relation:

```
R(A,B,C,D,E)
```

FD:

```
A → B
BC → D
D → E
```

### A+

শুরু:

```
A+ = {A}
```

A → B

```
A+ = {A,B}
```

BC → D প্রয়োগ করা যাবে না (C নেই)

সুতরাং,

```
A+ = {A,B}
```

---

### AC+

শুরু:

```
AC+ = {A,C}
```


```
AC+ = {A,B,C} [ Bcz, A → B ]
```


```
AC+ = {A,B,C,D} [ Bcz, BC → D ]
```


```
AC+ = {A,B,C,D,E} [Bcz, D → E ]
```

অর্থাৎ,

```
AC+ = {A,B,C,D,E}
```

সব attribute পাওয়া গেছে।

তাই,

```
AC = Candidate Key
```

```

```

- Transitivity
