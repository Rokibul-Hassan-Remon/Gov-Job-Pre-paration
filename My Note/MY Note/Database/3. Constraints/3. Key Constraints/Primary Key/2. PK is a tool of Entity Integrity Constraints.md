**Confusion:**

**"যদি Foreign Key, Referential Integrity enforce করার একটি টুল হয়, তাহলে Primary Key কী? এটি কোন Integrity Rule বা Constraint enforce করার জন্য ব্যবহৃত হয়?"**

চমৎকার প্রশ্ন। একই যুক্তি Primary Key-এর ক্ষেত্রেও প্রযোজ্য।

যদি আমরা **Relational Theory** এবং **SQL Implementation** আলাদা করি, তাহলে বিষয়টি পরিষ্কার হয়ে যাবে।

---

## Relational Theory Level

Relational Model-এ একটি Rule আছে:

### Entity Integrity Constraint

Rule:

> Every relation (table)-এর প্রতিটি tuple (row)-কে uniquely identify করা সম্ভব হতে হবে, এবং Primary Key NULL হতে পারবে না।

এটিই **Entity Integrity Constraint**।

---

## SQL Implementation Level

SQL-এ আমরা লিখি:

```sql
PRIMARY KEY (StudentId)
```

এটি DBMS-কে বলে:

> "Entity Integrity Rule enforce করো।"

---

## Parallel Comparison

|Theoretical Rule|SQL Mechanism|
|---|---|
|Entity Integrity Constraint|PRIMARY KEY Constraint|
|Referential Integrity Constraint|FOREIGN KEY Constraint|

---

### Entity Integrity

Rule:

```text
Every row must have a unique, non-null identifier.
```

Enforcement:

```sql
PRIMARY KEY
```

---

### Referential Integrity

Rule:

```text
Every foreign key value must reference an existing parent row.
```

Enforcement:

```sql
FOREIGN KEY
```

---

## তাহলে PRIMARY KEY আসলে কী?

Theory অনুযায়ী:

```text
PRIMARY KEY Constraint
```

হলো **Entity Integrity Constraint-এর enforcement mechanism**।

ঠিক যেমন:

```text
FOREIGN KEY Constraint
```

হলো **Referential Integrity Constraint-এর enforcement mechanism**।

---

## কিন্তু একটা সূক্ষ্ম পার্থক্য আছে

Foreign Key এবং Referential Integrity-এর নাম আলাদা।

কিন্তু:

```text
Entity Integrity Constraint
```

enforce করার জন্য SQL-এও

```text
PRIMARY KEY Constraint
```

নাম ব্যবহার করা হয়।

তাই অনেক বইয়ে মনে হয় Primary Key-ই Rule।

আসলে:

```text
Entity Integrity = Rule
Primary Key = SQL declaration that enforces the rule
```

---

## Viva Answer

> In relational theory, the Primary Key is associated with the Entity Integrity Constraint. The Entity Integrity Constraint is the rule that every row must be uniquely identifiable and its key cannot be NULL. The PRIMARY KEY constraint in SQL is the mechanism used to enforce that rule, just as a FOREIGN KEY constraint is used to enforce Referential Integrity.
