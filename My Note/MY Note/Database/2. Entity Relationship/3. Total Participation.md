**Total Participation** হলো ER Model-এর একটা constraint।

এর মানে:

> Entity set-এর **প্রতিটি row/entity-কে relationship-এ অংশগ্রহণ করতেই হবে**।

অর্থাৎ relationship optional না, mandatory।

---

## Example 1: Employee ↔ Department

ধরো Business Rule:

> প্রত্যেক Employee অবশ্যই একটি Department-এর অধীনে থাকবে।

ERD:

```text
Department -------- Employee
           works_in
```

এখানে:

- Employee-এর Total Participation আছে।
    
- Department-এর Total Participation নেই।
    

কারণ:

✅ প্রতিটি Employee-এর Department লাগবেই।

❌ কিন্তু নতুন Department থাকতে পারে যেখানে এখনো Employee নেই।

---

## Database View

```sql
Employee
---------
EmployeeId
Name
DepartmentId NOT NULL
```

এখানে:

```sql
DepartmentId NOT NULL
```

Total Participation enforce করছে।

---

## Example 2: Student ↔ University

Business Rule:

> প্রতিটি Student অবশ্যই একটি University-তে ভর্তি থাকবে।

```text
Student -------- EnrolledIn -------- University
```

Student side = Total Participation

কারণ Student entity থাকলে অবশ্যই University reference থাকবে।

# Partial Participation

Business Rule:

> একজন Employee-এর Parking Space থাকতে পারে, নাও থাকতে পারে।

```text
Employee -------- Assigned -------- ParkingSpace
```

এখানে Employee-এর Participation Partial।

কারণ:

```text
Employee
 ├─ Parking আছে
 └─ Parking নেই
```

দুইটাই valid।

# ERD Symbol

Chen Notation-এ:

### Total Participation

```text
Employee ══ WorksIn ── Department
```

Double Line।

### Partial Participation

```text
Employee ── Assigned ── Parking
```

Single Line।

# System Design Perspective

বাস্তব Software Design-এ Total Participation সাধারণত translate হয়:

### 1. NOT NULL Foreign Key

```sql
DepartmentId NOT NULL
```

---

### 2. Required Navigation Property

C# Example:

```csharp
public class Employee
{
    public int Id { get; set; }

    public int DepartmentId { get; set; }

    public Department Department { get; set; }
}
```

Employee Department ছাড়া exist করতে পারবে না।

---

### 3. Business Invariant

DDD/System Design-এ বলি:

> "An Employee cannot exist without a Department."

এটাই আসলে Total Participation-এর business meaning।

# Real Production Examples

### Total Participation

#### OrderLine → Order

```text
OrderLine ---- belongs_to ---- Order
```

OrderLine Order ছাড়া exist করতে পারে না।

---

#### Passport → Person

অনেক দেশে:

```text
Passport ---- owned_by ---- Person
```

Passport owner ছাড়া অর্থহীন।

---

#### InvoiceItem → Invoice

```text
InvoiceItem ---- belongs_to ---- Invoice
```

InvoiceItem একা থাকতে পারে না।

# Interview-Level Summary

**Total Participation = Every entity instance must participate in the relationship at least once.**

ERD View:

```text
Mandatory Relationship
```

Database View:

```text
Foreign Key NOT NULL
```

System Design View:

```text
Entity cannot exist without the related entity.
```

যেমন:

- Employee must belong to a Department.
    
- OrderLine must belong to an Order.
    
- InvoiceItem must belong to an Invoice.
    

এগুলো Total Participation-এর classic real-world examples।
